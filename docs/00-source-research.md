建立 ISEKAI ZERO 的公开产品画像，并把可以合法借鉴的功能、交互和商业系统转译为 Evolvria 的自有需求。本文不是素材抓取清单，也不是接口复刻方案，而是后续产品、设计、工程文档的事实基线。

## 产品定位观察

ISEKAI ZERO 是一个互动 AI 故事平台，公开定位集中在“创建沉浸式故事线、与 AI 角色聊天、让选择改变叙事”。它不是单纯 chatbot，也不是传统视觉小说；它把角色卡、故事线、场景、玩家 Persona、长程记忆、创作者收益和内容审核组合成一个 UGC 叙事平台。

Evolvria 应复刻的是这个组合模式：用户进入应用后直接看到可玩的故事和角色，通过详情页理解世界设定，再以某个身份进入聊天或视觉小说体验。与 ISEKAI ZERO 的关键差异必须体现在品牌、世界观、示例内容、视觉语言和工程架构上。Evolvria 的第一版不做大型 Web UGC 平台，而是先做本地优先 Tauri 2 应用，让玩家和创作者能在本机完成创建、游玩、保存、导入导出和 AI 配置。

2026-07-02 使用 Playwright 观察官网首屏时，页面根据访问环境展示中文 UI，左侧有 Home/Explore/Create/Chat/Store/Profile 类入口，主区域包含横向故事线 hero、媒体卡片、Visual Novel 标识、Casts、消息量/聊天量/互动量等统计、移动商店入口和免费/付费 Mana 余额。该观察只用于确认信息架构与功能组合，不应复制页面上的具体故事标题、角色名、图片或用户内容。

| 视觉小说 | 公开指南描述把故事推进变成更具画面、角色、语音的播放体验。 | Evolvria Beta 提供 Scene Mode：背景、立绘、字幕、旁白、音效、语音，MVP 先预留数据结构。 |
| Dungeon Mind | 公开指南把 DM 作为规则裁定与骰点机制。 | Evolvria 将其设计为 `Fate Engine`：独立规则层先输出可解释结果，再交给叙事模型润色。 |
| Creator Centre | 创作者创建角色、故事线、媒体、标签、可见性，并可参与收益机制。 | 本地创作优先，发布、收益、审核、统计延后；创作表单必须支持版本和草稿。 |
| Mana/Arcane | 公开信息显示存在免费/付费积分、模型成本和创作者收益。 | Evolvria 使用自有命名 `Spark`/`Core` 或 `Insight`/`Credit`；MVP 只做成本估算，不做支付。 |
| 内容分级 | 公开指南含内容分级、封面规范和社区规则。 | 内置 `SFW`、`M17`、`AdultLocked`、`Rejected` 状态；默认禁用成人内容和未成年人敏感内容。 |

公开 `llms.txt` 还显示其创作者收益以文本生成消费为核心，并公开提到 25% profit share。Evolvria 不能照搬该比例或宣传承诺；它只说明“创作者可按可审计消费/净收入参与分成”这一平台机制，具体比例、结算口径和税务处理必须由自有商业与法务方案决定。

## 技术形态观察

官网 HTML 显示其 Web 端是 Expo / React Native Web 风格的 SPA，并包含移动应用 deep link、App Store、Google Play、SEO alternate language、社交追踪脚本等。sitemap 暴露静态页面、角色、故事线、创作者索引，说明公开平台有较大 UGC 内容库和搜索需求。

这些信息只用于产品推断。Evolvria 不应抓取 sitemap 中的角色/故事线，不应调用 `api.isekai.world` 作为内容来源，也不应把原站公开接口写入代码。工程上选择 Tauri 2 + Vue 3 是为了做跨平台桌面/移动本地应用，而不是复用对方 Web 架构。

## 合法复刻边界

Evolvria 可以复刻“故事线市场 + 角色互动 + 创作者工具 + 叙事 AI + 可选商业化”的产品结构，但必须重做表达。具体要求：

- 文档中出现 ISEKAI ZERO 名称仅用于来源标注和竞品分析。
- UI 不做像素级克隆；可保留暗色、媒体卡片、沉浸式阅读这类通用模式，但重做颜色、间距、排版、图标和动效。
- 示例角色、故事、封面、标签和开场对白必须原创。
- 不内置或缓存对方图片、视频、音频、故事、角色说明、用户评价、搜索结果。
- 不把公开 API 当运行时依赖，不镜像对方内容库。
- 不复刻对方积分名、收益文案、政策文案或具体商业条款；只复刻“成本可见、创作者可分成、内容要审核”的机制。

## 风险

- 视觉相似度风险：暗色蓝紫和动漫卡片是行业常见表达，但需要通过自有色板、布局密度和组件语言拉开距离。
- UGC 法务风险：云端发布后会涉及版权侵权、成人内容、未成年人、骚扰、极端内容、退款和税务。
- AI 成本风险：视觉小说、语音、图片生成和长上下文模型会快速放大成本，必须提供预算上限和降级策略。
- 隐私风险：本地 Persona、聊天和创作草稿可能包含敏感信息，默认不上传，云同步必须显式授权。

## 关键决策

- Evolvria 先做本地优先 Tauri 2 MVP；Web UGC 平台、支付和收益后置。
- 原创品牌、原创示例内容、原创 UI token 是第一优先级，不追求页面像素一致。
- 数据模型可以参考公开功能形状，但字段命名和业务规则采用 Evolvria 自有定义。
- 所有外部来源只以链接和摘要形式保留，不复制长文或媒体。
