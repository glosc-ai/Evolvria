# 产品需求

## 目标

定义 Evolvria 的产品定位、目标用户、MVP 范围、非目标、成功指标和发布约束，让工程团队可以从文档直接进入设计和实现。

## 范围

- 自有品牌 AI 叙事应用，不复制 ISEKAI ZERO 品牌和内容。
- Tauri 2 跨平台应用，MVP 以桌面为主，移动端作为架构预留。
- 本地优先体验：无需账号也能创建、游玩、保存、导入和导出。
- 支持 mock AI 和 OpenAI-compatible provider；云端账号、UGC、支付和创作者收益作为后续阶段。

## 产品定位

Evolvria 是一个“可创作、可游玩、可保存的 AI 故事宇宙”。玩家像进入一本会回应自己的视觉小说，创作者像维护一个可运行的互动剧本。产品不只提供单次聊天，而是围绕故事线、角色、玩家 Persona、记忆、章节摘要和规则裁定形成长期体验。

一句话：用 Tauri 2 做一个本地优先的 AI 互动故事桌面应用，之后再扩展为云端 UGC 平台。

## 目标用户

| 用户 | 需求 | MVP 支持 |
| --- | --- | --- |
| AI 角色聊天用户 | 想快速进入一个角色或场景，获得稳定人设和记忆 | 精选故事、角色卡、聊天、重试、继续 |
| 互动小说玩家 | 想让选择影响剧情，不希望每次都从零开始 | 存档、章节摘要、Persona、故事状态 |
| TRPG/跑团爱好者 | 想要骰点、属性、判定和可解释结果 | MVP 预留，Beta Fate Engine |
| 轻量创作者 | 想创建角色和故事线，自己玩或分享给朋友 | 本地创作草稿、导入导出 |
| 高级创作者 | 想发布、获得统计、审核和收益 | Cloud 阶段 |

## 核心体验

1. 打开应用，看到继续游玩、推荐故事、最近创建和 AI provider 状态。
2. 进入内容库，按故事线、角色、标签、模式和分级筛选。
3. 打开故事详情，查看封面、简介、Casts、标签、支持的玩法、内容分级和成本提示。
4. 选择或创建 Persona，确认叙事偏好和安全边界。
5. 进入聊天或阅读界面，发送动作、对话或选择，AI 生成回复。
6. 应用自动保存消息、摘要、关系变化和关键事件。
7. 用户可重试、继续、回滚、导出存档或转到创作工作台修改设定。

## MVP 范围

MVP 必须交付：

- Tauri 2 + Vue 3 + Vite + Pinia 工程骨架。
- 暗色桌面主界面、响应式移动布局预留。
- 首页、内容库、故事详情、启动 Persona、聊天、创作草稿、存档、设置页面。
- 原创示例故事线、角色、封面占位和标签。
- `mock` provider：无网络也能生成可预测的叙事回复。
- OpenAI-compatible provider：用户配置 base URL、API key、model、上下文限制。
- 本地 JSON workspace：故事、角色、Persona、聊天、消息、摘要、资产元数据。
- 导入/导出 zip、自动备份、基本错误恢复。
- 成本估算 UI：token 估计、模型单价占位、预算上限。
- 内容分级字段、成人内容默认锁定、举报/审核数据结构预留。

## MVP 非目标

- 不实现 ISEKAI ZERO 的云端内容库、推荐算法、社交关注和创作者主页。
- 不实现真实支付、积分购买、创作者提现、税务和账单。
- 不实现完整视觉小说渲染、语音生成、图片生成、视频生成。
- 不实现多人实时协作或在线跑团。
- 不实现移动应用商店发布，但工程结构不阻断 iOS/Android。
- 不内置任何第三方版权角色、图片、故事或公开站点内容。

## Beta 范围

Beta 在 MVP 稳定后增加：

- Scene Mode 视觉小说：背景、角色立绘、字幕、旁白、场景状态。
- Fate Engine：属性、骰点、难度、后果、失败推进。
- 摘要章节与 Arc：长期记忆压缩、剧情阶段、关系图谱。
- 创作工作台增强：版本、媒体管理、校验、预览、模板。
- SQLite 可选迁移，提升大型内容库性能。
- 云同步技术预览：账号、设备、冲突解决、私有内容同步。

## 云端平台范围

云端阶段增加：

- UGC 发布、搜索、推荐、收藏、评分或互动统计。
- 创作者收益、积分消费、模型成本结算和账本。
- 审核后台、举报、申诉、分级锁、封面审核和风控日志。
- 服务端 AI 网关：统一密钥、模型路由、内容安全、限流和账单。
- Web 分享页和创作者公开主页。

## 成功指标

| 阶段 | 指标 |
| --- | --- |
| MVP | 首次启动 3 分钟内完成第一轮聊天；本地存档恢复成功率 99% 以上；mock 模式无网络可玩；导入导出通过手动验收。 |
| Beta | 单个故事连续 100 轮上下文不崩坏；摘要可解释；Scene Mode 30fps 基本流畅；Fate Engine 判定可复现。 |
| Cloud | 内容发布审核 SLA 可度量；同步冲突可恢复；支付和账本可追溯；举报处理有审计日志。 |

## 关键决策

- 第一版以“本地可玩闭环”为唯一发布标准，不等待云端平台。
- 没有 API key 的用户也必须能用 mock provider 体验完整流程。
- 所有用户数据默认留在本地；云同步必须显式启用。
- 内容安全字段从 MVP 开始建模，哪怕审核后台后置。
- 品牌和示例世界观使用 Evolvria 原创设定，避免被误认为 ISEKAI ZERO 客户端。

## 验收标准

- 产品范围能映射到 [功能映射](02-feature-parity-map.md) 和 [路线图](15-roadmap.md)。
- MVP 用户流覆盖发现、详情、启动、聊天、保存、创作草稿和设置。
- 每个非目标都在路线图中有明确“暂不做”或“后续阶段”解释。
- 工程团队可以据此拆分 Epic、Story 和验收测试。
