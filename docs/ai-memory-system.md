# AI 记忆系统

## 目标

AI 不能只依赖上下文窗口记住世界。Evolvria 需要把事实、状态、事件和摘要结构化保存，并在每次调用前检索相关内容。

## 记忆类型

- 世界设定记忆：世界规则、历史、势力、禁忌、地理。
- 角色记忆：角色亲历事件、目标变化、关系印象、秘密。
- 地点记忆：地点状态、发生过的事件、控制势力。
- 事件记忆：一次已发生事件的结构化记录。
- 玩家偏好记忆：叙事风格、禁用内容、玩家选择习惯。
- 摘要记忆：长时间线压缩后的阶段总结。

## 写入原则

每次 AI 返回后，系统要抽取并写入：

- 新事实。
- 状态变化。
- 关系变化。
- 重要承诺。
- 未解决悬念。
- 玩家明确表达的偏好。

不写入：

- 无意义寒暄。
- 重复描述。
- 未确认的猜测，除非标注为传闻或怀疑。

## 记忆字段

```json
{
  "id": "mem_001",
  "scope": "character",
  "owner_id": "char_hero",
  "text": "主角在黑石镇救下了艾琳。",
  "facts": ["主角救下艾琳", "地点是黑石镇"],
  "event_id": "evt_001",
  "importance": 0.8,
  "confidence": 0.95,
  "created_at": "world_time",
  "last_used_at": "world_time",
  "tags": ["relationship", "rescue", "blackstone"]
}
```

## 检索策略

每次 AI 调用前组合以下上下文：

- 当前场景状态：玩家位置、同行角色、当前目标。
- 最近事件：最近 5 到 15 条事件。
- 高相关记忆：按角色、地点、标签和关键词检索。
- 高重要记忆：长期目标、核心关系、世界规则。
- 当前世界摘要：压缩后的全局摘要。

MVP 可先使用关键词和标签检索。后续可加入 embedding 向量检索。

## 摘要策略

当事件历史超过阈值时生成摘要：

- 每 20 到 30 个事件生成一个阶段摘要。
- 每个角色维护独立摘要。
- 每个地点维护独立摘要。
- 摘要不删除原始事件，只用于降低上下文长度。

摘要必须包含：

- 已发生的重要事件。
- 角色目标变化。
- 关系变化。
- 未解决问题。
- 世界状态变化。

## 冲突处理

AI 可能产生与已有设定冲突的内容。处理优先级：

1. 玩家明确设定。
2. 结构化世界规则。
3. 已确认事件历史。
4. 角色记忆。
5. AI 新生成内容。

冲突时可以：

- 拒绝该状态变更。
- 要求 AI 重新生成。
- 记录为误传、幻觉或角色谎言，但不能直接覆盖事实。

## Prompt 结构

建议顺序：

1. 系统规则：保持一致性、返回 JSON、不要覆盖已确认事实。
2. 世界摘要。
3. 当前场景。
4. 相关角色卡。
5. 相关地点卡。
6. 最近事件。
7. 检索到的长期记忆。
8. 玩家行动。
9. 输出格式要求。
