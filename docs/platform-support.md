# 跨平台支持

## 目标

桌面端、移动端和平板端共享同一套世界状态、AI 编排、存档格式和核心玩法。平台差异集中在布局、输入、文件能力和 native 命令。

## 当前实现

- Tauri 2 桌面能力已接入。
- Vue 运行在浏览器时可使用 `localStorage` fallback。
- UI 使用响应式布局：桌面固定左侧导航，移动端顶部栏 + 抽屉 + 底部导航。
- `get_platform_capabilities` 可返回 OS、是否移动端、是否可 reveal 目录、是否可分享文件、是否可使用文件选择器和 app data dir。
- 移动端 scaffold 命令保留，但 Android/iOS 签名和发布不在当前 MVP。

## 支持平台目标

优先级：

1. 桌面端开发版：macOS、Windows、Linux。
2. 平板端体验版：长文本、地图、人物资料并列。
3. 移动端体验版：小屏输入、查看故事、短行动。

当前验证重点是桌面和浏览器响应式布局。

## 共享能力

所有平台应一致支持：

- 新建世界。
- 本地或远端 AI 世界扩写。
- 玩家行动输入。
- 推荐行动选择。
- 时间线、角色、地点、线索和世界观查看。
- 本地存档。
- Glosc One 配置和用量提示。
- AI 失败后的恢复。

地图图片导入和系统分享属于后续平台 UI 补齐项。

## 桌面端规格

当前：

- 固定左侧导航。
- 主内容区响应式扩展。
- 支持键盘输入。
- Tauri active 存档、备份、checkpoint、zip 导出。
- 可通过 native 能力 reveal/share path。

目标：

- 文件选择器导入 JSON/zip/map 图片。
- 地图滚轮缩放和拖拽。
- 更高信息密度的时间线和日志视图。

## 移动端规格

当前：

- 顶部栏。
- 抽屉导航。
- 底部 5 项主导航：探索、地图、人物、地点、时间线。
- safe-area padding。

目标：

- 软键盘不遮挡输入框。
- 地图双指缩放和拖动。
- 系统分享导出 zip。
- 系统文件选择导入。

## 平板端规格

当前：

- 依赖 CSS grid 和 Tailwind 断点自然适配。

目标：

- 横屏双栏/三栏。
- 地图与地点详情并列。
- 创建世界表单与预览并列。
- 分屏窗口下退回单栏。

## 响应式断点

当前主要使用 Tailwind 断点：

- 默认：移动单栏。
- `sm`：增加水平 padding。
- `lg`：启用固定侧栏布局。
- `xl`：部分页面启用主内容 + 右侧上下文。

文档建议断点仍可作为设计目标：

- 小屏：小于 700px。
- 中屏：700px 到 1099px。
- 大屏：1100px 以上。

## 平台不变量

- 世界状态和存档 schema 不因平台变化。
- AI 请求结构不因平台变化。
- 文件路径和分享能力只在 service/native 层处理。
- UI 可因平台调整布局，但不能隐藏核心流程。
