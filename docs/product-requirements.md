# 产品需求

## 一句话定位

Evolvria 是一个 AI 驱动、本地优先的叙事/世界模拟游戏。玩家定义初始角色和世界观后，系统持续记录事件、记忆、关系、地点、地图和线索；配置 Glosc One 后由远端模型增强叙事，未配置时使用本地 mock 完整运行。

## 目标玩家

- 喜欢文字冒险、TRPG、开放世界 RPG、角色扮演和世界观构建的玩家。
- 希望自己设定角色关系、地点和世界规则，再让 AI 帮助推进故事的人。
- 重视本地存档、可导出数据和可恢复探索历史的用户。
- 可接受 AI 调用可能消耗 Glosc One 额度，但希望离线或未配置时仍能试用核心流程的用户。

## 核心价值

- 世界状态结构化保存，不把聊天记录当作唯一记忆。
- 玩家行动会写入时间线、记忆、关系和线索进度。
- NPC 和线索不是纯装饰，后续可通过 tick 和事件持续推进。
- 地图以地点和路线参与移动与上下文构建。
- 桌面、移动和平板共享同一存档 schema。

## 当前 MVP 已具备

- 新建世界五步表单：世界、主角、关键角色、内容偏好、预览。
- 本地生成 schema v1 世界：角色、地点、势力、地图、开局事件、记忆、线索和推荐行动。
- 玩家自由输入或点击推荐行动。
- Glosc One 远端调用；未配置时本地 mock。
- 玩家行动写入时间线、记忆、关系、地点状态和线索进度。
- 本地 NPC tick 生成简短非玩家事件。
- 地图查看、隐藏未知地点、移动、手动加地点、加路线。
- 人物、地点、时间线、线索、世界观、存档和设置页面。
- Tauri 本地存档、备份、AI checkpoint 和 zip 导出。
- 浏览器开发环境 `localStorage` fallback。
- Vitest 领域测试和 Playwright 主流程渲染测试。

## 当前 MVP 暂未具备

- 远端 `world_expand` 直接返回完整结构化世界并被解析。
- 地图图片导入/生成的 Vue UI 入口。
- 原生文件选择导入 zip。
- 自动摘要任务。
- 自动重试和请求取消。
- Keychain/Keystore 级别安全存储。
- 云同步。
- 复杂战斗、经济、制作、交易系统。
- 多人在线或 UGC 市场。
- AI 自动生成图片、语音或视频。

## 成功标准

- 玩家可在 10 分钟内创建世界并进入探索。
- 未配置 Glosc One 时，仍可完成创建、探索、地图、存档和重进。
- 配置错误或远端响应不可用时，玩家行动不会破坏存档。
- 连续 10 轮玩家行动后，timeline、memory、thread 和 location 引用保持一致。
- 退出并重新进入后，active 世界完整恢复。
- 桌面宽屏、桌面窄屏、手机竖屏和平板横屏都能完成核心流程。

## 非目标

- 不追求所有内容完全由 AI 即兴生成。核心状态必须结构化保存。
- 不把完整历史无差别发送给 AI。
- 不在客户端明文硬编码任何服务密钥。
- 不在 MVP 中实现云端账号和在线多人。

## 商业模式

客户端免费。AI 调用通过用户自己的 Glosc One 配置消耗额度。游戏内需要清晰说明：

- 哪些行为会调用远端 AI。
- 未配置 Glosc One 时会使用本地模拟。
- 每次调用的大致 token 消耗和风险。
- 失败后的降级、重试或恢复策略。
