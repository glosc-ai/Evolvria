# 安全、隐私与内容策略

## API 凭据

- 不得把 Glosc One Key 写入仓库。
- 不得硬编码用户密钥。
- 日志中必须脱敏 Authorization、token、cookie 等字段。
- 当前版本设置会通过 Tauri 保存到 `settings.json`；浏览器开发环境 fallback 到 `localStorage`。
- 保存非空 Key 前，用户必须勾选本机存储风险确认。

设置页风险文案与实现一致：

> 当前版本会优先使用平台安全存储；若平台不可用，会把访问令牌保存在本机应用数据目录的 settings.json。请只在可信设备上保存，并避免上传或分享该文件。

注：当前代码未接入系统 Keychain/Keystore；“平台安全存储”是目标描述，不应在发布说明中宣称已完成。

## 玩家数据

玩家输入、世界设定、AI 日志和存档都可能包含私人信息。默认策略：

- active 存档只保存在用户设备。
- 只有远端 AI 调用所需上下文会发送到 Glosc One。
- 未配置 Glosc One 时不发送网络请求。
- 导出 zip 或 JSON 时应提醒其中包含玩家生成内容和 AI 日志。

当前 UI 尚未提供删除世界、清空 AI 日志、清空所有本地数据的专门入口；这些属于发布前隐私功能待办。

## AI 上下文最小化

当前玩家行动上下文只发送：

- `world`
- 当前地点。
- 同行角色 ID。
- 附近地点。
- 检索到的记忆。
- 最近 8 条事件。

不应无条件发送完整 timeline、完整 memories 或 Glosc token。

## 日志策略

`log_level` 支持：

- `default`
- `debug`
- `deep`

当前 `AIRequestLog` 默认记录 purpose、hash、model、时间、状态、错误、usage 和 summary。`raw_response` 是可选字段，写入前必须调用脱敏逻辑。

深度日志导出前必须提示可能包含隐私内容。当前 UI 尚未提供日志导出入口。

## 内容边界

玩家可在创建世界和设置中输入内容偏好。系统和 prompt 应遵守：

- 尊重玩家内容偏好和禁用内容。
- 避免生成违法或明显有害内容。
- 对敏感内容使用淡出、概述或替代叙事。
- 不生成针对现实个人的隐私侵犯内容。
- 不把未公开秘密泄露给不应知道该秘密的角色。

## 存档完整性

- Tauri 端写入使用临时文件和原子替换。
- active 存档覆盖前创建最多 5 个备份。
- AI 请求前创建 checkpoint。
- 加载时校验 schema。
- 损坏存档不应导致应用崩溃。
- 导入外部 JSON/zip 时只解析数据，不执行脚本或动态代码。

## 发布前隐私待办

- 删除 active 世界和备份入口。
- 导出前隐私确认。
- Keychain/Keystore 级别安全存储。
- 连接测试不记录 Key。
- deep 日志二次确认和自动过期。
