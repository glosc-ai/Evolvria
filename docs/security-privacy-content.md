# 安全、隐私与内容策略

## API 凭据

- 不得把 Glosc One 凭据写入仓库。
- 不得硬编码服务密钥。
- 本地保存凭据时优先使用平台安全存储。
- 日志中必须脱敏 Authorization、token、cookie 等字段。

## 玩家数据

玩家输入可能包含私人信息。默认策略：

- 本地存档只保存在用户设备。
- 只有 AI 调用所需上下文会发送到 Glosc One。
- 提供清除 AI 日志和删除世界的入口。
- 导出存档时提醒其中包含玩家生成内容。

## AI 上下文最小化

每次请求只发送必要内容：

- 当前场景。
- 相关角色。
- 相关地点。
- 最近事件。
- 检索到的记忆。

不要无条件发送完整历史。

## 内容边界

玩家可设置禁用内容。系统 Prompt 应包含：

- 尊重玩家内容偏好。
- 避免生成违法或明显有害内容。
- 对敏感内容使用淡出或替代叙事。
- 不生成针对现实个人的隐私侵犯内容。

## 日志策略

调试日志分级：

- 默认：只记录请求 ID、类型、状态、用量。
- 调试：记录 Prompt 摘要和响应摘要。
- 深度调试：记录完整 Prompt 和响应，需用户手动开启。

深度调试日志导出前应提示可能包含隐私。

## 存档完整性

- 存档写入使用临时文件和原子替换。
- 加载时校验 schema。
- 损坏文件不应导致应用崩溃。
- 导入外部存档时不执行任何脚本或动态代码。
