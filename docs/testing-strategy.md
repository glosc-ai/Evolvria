# 测试策略

## 目标

Evolvria 的风险包括代码崩溃、AI 输出不可解析、世界状态混乱、记忆丢失、存档损坏和响应式 UI 退化。测试需要覆盖领域逻辑、主流程渲染、native 命令和手动跨平台验收。

## 当前自动化测试

Vitest：

- `tests/domain.test.ts`
- 覆盖世界创建、schema 校验、patch 防覆盖、玩家行动、记忆检索、时间线筛选、地图标注、路线、用量估算和 AI 日志汇总。

Playwright：

- `tests/e2e/app.spec.ts`
- 覆盖首页、新建世界、创建世界、探索、地图、人物、时间线主流程。

Rust：

- `src-tauri/src/lib.rs` 内有 `chat_endpoint` 和 payload schema 校验单测。

## 常用命令

```bash
yarn typecheck
yarn test
yarn test:ui

cd src-tauri && cargo test
```

## 单元测试优先级

优先覆盖：

- ID 生成。
- schema 校验。
- 世界创建默认值。
- patch 校验和应用。
- 时间推进。
- 事件引用一致性。
- 记忆检索。
- 地图地点/路线。
- AI 用量估算。
- 存档导入校验。

## 集成测试场景

应逐步补齐：

- 创建世界后保存并重新加载。
- 玩家行动成功后写入事件、记忆、关系和线索。
- 远端响应不可用时使用本地 mock。
- AI checkpoint 在请求前保存。
- 导出 zip 后包含 manifest 和 payload。
- 导入损坏 JSON 时有错误提示。
- 设置保存 Key 前必须确认风险。

## AI 模拟测试

自动测试不得依赖真实 Glosc One。应使用固定 mock 或 fixture 覆盖：

- 正常世界扩写。
- 正常玩家行动。
- 远端响应缺字段。
- 远端非 JSON。
- `status = error`。
- patch 试图覆盖玩家确认字段。
- 引用不存在角色/地点。

当前 repo 还没有 `tests/fixtures/ai`，如引入 fixture 目录需同步文档和测试。

## 叙事一致性测试

规则建议：

- 死亡或离队角色不能继续同行。
- NPC 不知道未公开秘密。
- 地点毁坏后不能仍描述为完好。
- 已确认关系不能无原因重置。
- 时间线不能倒序。
- 已知地点坐标不能被 AI 改写。

当前仅实现部分引用检查，其他规则可在后续迭代中测试驱动补齐。

## 手动验收清单

每次发布前检查：

- 新建世界流程可完成。
- 未配置 Glosc One 时可以探索。
- 配置错误时不丢存档。
- 连续 10 轮玩家行动不中断。
- 存档退出重进后状态正确。
- 地图标注和路线重进后仍存在。
- 存档导出可以生成路径或 Blob URL。
- JSON 文本导入会校验 schema。
- 桌面宽屏、桌面窄屏、手机竖屏和平板横屏布局不重叠。

## 性能检查

关注：

- 长时间线列表滚动。
- 地图缩放。
- AI 等待时 UI 是否卡死。
- 大存档加载速度。
- 移动端软键盘弹出后的输入框位置。
