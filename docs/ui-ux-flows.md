# UI/UX 流程

## 信息架构

当前路由：

- 首页
- 引导
- 新建世界
- 探索
- 地图
- 人物
- 地点
- 时间线
- 线索
- 世界观
- 存档
- 设置

桌面端通过左侧导航进入；移动端通过顶部菜单、抽屉和底部主导航进入。

## 首页

首页展示：

- 产品标题和一句说明。
- 继续游戏。
- 新建世界。
- 存档列表。
- 设置。
- 当前世界统计：角色、地点、事件、线索。

没有 active 世界时，继续游戏会提示错误。

## 新建世界流程

当前为 5 步：

1. 世界名称、类型、基调。
2. 主角姓名、身份描述、目标、能力和弱点；可用“智能生成”调用设置中的 AI 大模型，并通过游戏内 skills 补全角色卡。
3. 关键角色列表，可新增角色；单个角色可用“智能生成”补全身份、目标、秘密、行动倾向和外貌描述；未配置模型或调用失败时不使用模板兜底。
4. 内容偏好、叙事详细度、NPC 自主频率。
5. 预览和 AI 用量估算。

创建成功后进入探索页。

## 探索界面

核心区域：

- 当前地点。
- 当前世界时间。
- AI 生成状态。
- 最近叙事。
- 推荐行动。
- 玩家输入框。

右侧上下文：

- 当前地点描述。
- 快速前往其他地点。
- 最近事件。
- 同行角色。

AI 调用期间禁用输入和推荐行动按钮。

## 地图界面

当前显示：

- SVG 地形底图。
- 创世生成的地区色块。
- 地点和名称。
- 路线。
- 当前地点高亮。
- 地点详情。
- 创世地图锁定状态。

交互：

- 缩放。
- 显示/隐藏未知地点。
- 点击地点。
- 移动到地点。

后续：

- 图片底图。
- 地图查看层级和更丰富的地区筛选。
- NPC 位置。

## 人物界面

展示角色：

- 名称和身份。
- 可见性/是否同行。
- 当前地点。
- 性格、目标、秘密。
- 关系。
- 外貌描述和角色形象。
- 玩家备注。
- 重新生成或修改角色形象；未填写外貌时根据角色卡自动补全提示词。

角色过滤：

- 全部。
- 同行。
- 仅听闻。
- 敌对/竞争。

未知信息应避免提前剧透；当前数据层已有 `visibility`，UI 后续应更严格地隐藏 hidden 信息。

## 地点界面

展示地点：

- 名称、类型、描述。
- 可见性。
- 控制势力。
- 状态标签。
- 玩家备注。

可编辑地点备注。

## 时间线界面

支持：

- 按类型筛选。
- 按角色筛选。
- 按地点筛选。
- 查看事件标题、描述、时间、参与者和结果。

事件应按世界时间和写入顺序展示。

## 线索界面

展示 thread：

- 标题。
- 描述。
- 类型。
- 状态。
- 优先级。
- 标签。
- 进度。

玩家可将 open thread 标记为 resolved。

## 世界观界面

展示：

- 世界名称、类型、基调。
- 世界摘要。
- 规则。
- 主题。
- 内容限制。
- 势力。
- 一致性检查结果。

## 存档界面

当前功能：

- 刷新存档列表。
- 导出当前世界。
- 查看 active、backup、ai_checkpoint 条目。
- 显示 schema 状态。
- 从 JSON 文本导入。

后续：

- 文件选择导入 zip。
- 删除备份。
- 恢复指定备份。
- reveal/share 导出路径。

## 设置界面

包含：

- Glosc One 服务地址、模型、访问 Key、超时。
- 本机 Key 存储风险确认。
- AI 调用前确认。
- 用量估算。
- 自动保存。
- 自动重试。
- 日志级别。
- 字体大小。
- 内容偏好。
- 保存和重置。

## 文案原则

- 明确告诉玩家是否会消耗远端额度。
- AI 失败时给出可行动的下一步。
- 不暴露堆栈信息。
- 对长文本分段，提高小屏可读性。
- 对尚未接入的 native 能力，不在 UI 中承诺已可用。
