# 世界模拟系统

## 目标

世界不应只在玩家行动时变化。角色、地点、势力和线索需要随时间推进产生事件历史。当前实现提供了基础事件、时间推进、线索进度和本地 NPC tick；更复杂的模拟仍属后续。

## 当前已实现

- 玩家行动会根据 `time_delta_minutes` 推进世界时间。
- 每次玩家行动生成 timeline event。
- 玩家行动可写入角色/地点 patch、关系变化和记忆。
- open thread 会自动推进最多两个进度条目。
- `runNpcTick` 会选择一个 active NPC，写入 `npc_simulation` 事件和 AI 日志。
- `validateWorldConsistency` 检查事件引用的地点和角色是否存在。

## 时间推进

时间推进来源：

- 玩家行动。
- 玩家移动。
- 本地 NPC tick。
- 后续的休息、等待、长距离旅行和离线推进。

当前时间模型：

```json
{
  "day": 1,
  "hour": 8,
  "calendar_label": "第一纪元 1001 年 春"
}
```

`advanceWorldTime` 以小时为单位推进，分钟向下折算为小时；小于 60 分钟不会改变 hour。

## NPC 行动决策

当前本地 tick：

1. 找到第一个非主角且 `status = active` 的角色。
2. 生成简短事件描述。
3. 使用角色当前位置作为事件地点。
4. 如果角色可见性是 `met`，事件对玩家可见；否则为 hidden。
5. 写入 AI 日志，purpose 为 `npc_simulation`。

后续真实决策输入：

- 当前地点。
- 性格。
- 目标。
- 行动倾向。
- 关系网络。
- 最近记忆。
- 世界事件压力。

## 事件重要度

建议按 `importance` 区分：

- 低重要度：只更新位置或状态。
- 中重要度：写结构化事件和简短描述。
- 高重要度：调用 AI 生成叙事，并可能通知玩家。

高重要度条件：

- 涉及关键角色。
- 改变重要关系。
- 改变地点控制权。
- 造成失踪、背叛、发现或重大冲突。
- 影响玩家目标或 open thread。

## 世界一致性

必须遵守：

- 事件引用的地点和角色存在。
- NPC 不能同时出现在两个地点。
- 已死亡、失踪、被囚禁角色不能继续普通行动。
- 已知地点坐标和描述不能被 AI 覆盖。
- 世界当前时间只能由系统推进。
- 角色不能知道自己没有亲历或被告知的信息。

当前自动检查只覆盖引用存在性；其他规则需要后续实现。

## 离线推进

当前不做真实后台或离线推进。

后续设计：

- 记录最后活跃时间。
- 将现实离线时间映射为有限世界时间。
- 生成少量关键变化。
- 进入游戏时展示“离开期间发生了什么”摘要。
- 用户可在设置中关闭。

## 势力模拟

当前势力只记录：

- 名称。
- 目标/agenda。
- 态度。
- 控制地点。

后续可扩展：

- 资源。
- 敌友关系。
- 军事/贸易行动。
- 地点控制权变化。
- 与 thread 绑定的势力目标。
